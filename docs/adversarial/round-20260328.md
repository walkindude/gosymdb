# Adversarial Round 2026-03-28

## Scope

Black-box testing against `gosymdb 0.1.0` using only `--help` plus synthetic Go code in `testbench/22_alias_generic_consistency`.

## Findings

### 1. `implementors` misses interface type aliases

- Severity: high
- Synthetic code: `testbench/22_alias_generic_consistency`
- Reproduction:
  - `gosymdb index --root ./testbench/22_alias_generic_consistency --db /tmp/test.sqlite --force`
  - `gosymdb implementors --db /tmp/test.sqlite --iface 'testbench/alias_generic_consistency/alias.ReaderAlias' --json`
- Expected:
  - `alias.ReaderAlias` is a type alias of `iface.Reader`, so its implementors should include `alias.File`.
- Actual:
  - `implementors` returns an empty list.
- Why this is hard:
  - Aliases preserve identity at the language level, but indexers often normalize one path in one table and the other path in another table, which silently breaks joins.

### 2. `implementors` misses instantiated generic interfaces

- Severity: high
- Synthetic code: `testbench/22_alias_generic_consistency`
- Reproduction:
  - `gosymdb index --root ./testbench/22_alias_generic_consistency --db /tmp/test.sqlite --force`
  - `gosymdb implementors --db /tmp/test.sqlite --iface 'testbench/alias_generic_consistency/iface.Box[int]' --json`
- Expected:
  - `alias.GenericBox[int]` implements `iface.Box[int]`, so the result should include `alias.GenericBox`.
- Actual:
  - `implementors` returns an empty list.
- Why this is hard:
  - Generic method sets and instantiated interface names require consistent normalization of type parameters across symbol, implements, and query layers.

### 3. Call edges target interface methods that are not definable symbols

- Severity: medium
- Synthetic code: `testbench/22_alias_generic_consistency`
- Reproduction:
  - `gosymdb index --root ./testbench/22_alias_generic_consistency --db /tmp/test.sqlite --force`
  - `gosymdb callees --db /tmp/test.sqlite --symbol 'testbench/alias_generic_consistency/use.UseReader' --json`
  - `gosymdb def --db /tmp/test.sqlite 'testbench/alias_generic_consistency/iface.Reader.Read' --json`
  - `gosymdb callees --db /tmp/test.sqlite --symbol 'testbench/alias_generic_consistency/use.UseBoxInt' --json`
  - `gosymdb def --db /tmp/test.sqlite 'testbench/alias_generic_consistency/iface.Box[int].Unbox' --json`
- Expected:
  - Any callee fqname emitted by `callees` should round-trip through `def` or `find`.
- Actual:
  - `callees` emits `iface.Reader.Read` and `iface.Box[int].Unbox`, but `def` reports `symbol: null` for both.
- Why this is hard:
  - Interface-dispatch modeling is legitimately approximate, but once the CLI surfaces a fqname as a concrete query target it has to remain internally consistent across commands.

## Additional Findings

### 4. `blast-radius --pkg` returns callers outside the requested package prefix

- Severity: high
- Synthetic code: `testbench/23_blast_pkg_filter`
- Reproduction:
  - `gosymdb index --root ./testbench/23_blast_pkg_filter --db /tmp/test.sqlite --force`
  - `gosymdb callers --db /tmp/test.sqlite --symbol 'testbench/blast_pkg_filter/core.Target' --pkg 'testbench/blast_pkg_filter/use' --json`
  - `gosymdb blast-radius --db /tmp/test.sqlite --symbol 'testbench/blast_pkg_filter/core.Target' --pkg 'testbench/blast_pkg_filter/use' --json`
- Expected:
  - Both commands should restrict results to callers whose package path starts with `testbench/blast_pkg_filter/use`, so the result set should include `use.Use` and exclude `core.Local`.
- Actual:
  - `callers` respects the filter, but `blast-radius` leaks `core.Local`.
- Why this matters:
  - Package scoping is the only practical way to limit blast radius in large monorepos. A leaky filter makes impact reports misleading and hard to trust.

### 5. `gosymdb help` fails in a fresh directory because it still requires a database

- Severity: medium
- Synthetic code: none required
- Reproduction:
  - `mkdir /tmp/gosymdb-help-empty && cd /tmp/gosymdb-help-empty`
  - `gosymdb help`
- Expected:
  - The built-in help command should print help and exit successfully without requiring any database.
- Actual:
  - The command exits with `no database found ...`.
- Why this matters:
  - The prompt flow for a first-time user is to ask for help before indexing anything. Failing there is avoidable friction and contradicts normal Cobra CLI expectations.

### 6. `implementors` accepts `--iface` and `--type` together instead of rejecting the ambiguous mode

- Severity: medium
- Synthetic code: any indexed database
- Reproduction:
  - `gosymdb implementors --db /tmp/test.sqlite --iface 'iface.Doer' --type 'RealDoer'`
- Expected:
  - The command should reject the invocation and require exactly one of `--iface` or `--type`.
- Actual:
  - It runs and returns a merged result set.
- Why this matters:
  - The command is presented as two opposite query directions. Accepting both silently makes the result semantics unclear and hard to automate against.

## UX Friction

- `gosymdb help` failing on a fresh machine is a poor first-run experience.
- The `implementors` help text reads like an either/or mode switch, but the command does not enforce that contract.
- Several commands silently normalize invalid input instead of rejecting it, which makes it hard to distinguish тАЬempty resultтАЭ from тАЬbad queryтАЭ.

## Inconsistencies

- `callers --pkg` respects the caller package filter, but `blast-radius --pkg` does not.
- The CLI advertises `help` as a normal subcommand in `--help`, but invoking it behaves differently from `--help`.

## Feature Requests

- Add explicit validation for enum-like flags such as `--kind` and `--ref-kind` so typos fail loudly.
- Add a dedicated `--from-pkg` or `--caller-pkg` flag to `blast-radius` if the current `--pkg` flag is intended to mean something else.

## What Worked Well

- Indexing small standalone modules was fast and deterministic.
- `callers --pkg` behaved as expected and provided a good control query for the blast-radius regression.
- The JSON output shape is generally easy to assert against from tests.
