# Adversarial Round 2026-03-28

## Scope

Black-box testing against `gosymdb 0.1.0` using only `--help` plus synthetic Go code in `testbench/22_alias_generic_consistency`.

## Findings

### 1. `implementors` misses interface type aliases

- Severity: high
- Synthetic code: `testbench/22_alias_generic_consistency`
- Reproduction:
  - `gosymdb index --root ./testbench/22_alias_generic_consistency --db /tmp/test.sqlite --force`
  - `gosymdb implementors --db /tmp/test.sqlite --iface 'testbench/alias_generic_consistency/alias.ReaderAlias' --json`
- Expected:
  - `alias.ReaderAlias` is a type alias of `iface.Reader`, so its implementors should include `alias.File`.
- Actual:
  - `implementors` returns an empty list.
- Why this is hard:
  - Aliases preserve identity at the language level, but indexers often normalize one path in one table and the other path in another table, which silently breaks joins.

### 2. `implementors` misses instantiated generic interfaces

- Severity: high
- Synthetic code: `testbench/22_alias_generic_consistency`
- Reproduction:
  - `gosymdb index --root ./testbench/22_alias_generic_consistency --db /tmp/test.sqlite --force`
  - `gosymdb implementors --db /tmp/test.sqlite --iface 'testbench/alias_generic_consistency/iface.Box[int]' --json`
- Expected:
  - `alias.GenericBox[int]` implements `iface.Box[int]`, so the result should include `alias.GenericBox`.
- Actual:
  - `implementors` returns an empty list.
- Why this is hard:
  - Generic method sets and instantiated interface names require consistent normalization of type parameters across symbol, implements, and query layers.

### 3. Call edges target interface methods that are not definable symbols

- Severity: medium
- Synthetic code: `testbench/22_alias_generic_consistency`
- Reproduction:
  - `gosymdb index --root ./testbench/22_alias_generic_consistency --db /tmp/test.sqlite --force`
  - `gosymdb callees --db /tmp/test.sqlite --symbol 'testbench/alias_generic_consistency/use.UseReader' --json`
  - `gosymdb def --db /tmp/test.sqlite 'testbench/alias_generic_consistency/iface.Reader.Read' --json`
  - `gosymdb callees --db /tmp/test.sqlite --symbol 'testbench/alias_generic_consistency/use.UseBoxInt' --json`
  - `gosymdb def --db /tmp/test.sqlite 'testbench/alias_generic_consistency/iface.Box[int].Unbox' --json`
- Expected:
  - Any callee fqname emitted by `callees` should round-trip through `def` or `find`.
- Actual:
  - `callees` emits `iface.Reader.Read` and `iface.Box[int].Unbox`, but `def` reports `symbol: null` for both.
- Why this is hard:
  - Interface-dispatch modeling is legitimately approximate, but once the CLI surfaces a fqname as a concrete query target it has to remain internally consistent across commands.
